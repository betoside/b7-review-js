#17: Introdução a Programação Funcional

    a grosso modo, qdo se faz o código usando funções
    onde uma tem uma responsabilidade de processamento
    em seguida passa dados para outra, ou funções se comunicando
    para resolver coisas.

    na programação funcional não temos classes, temos funções. quando criamos
    o objeto já é ele mesmo que vamos manipular. mesmo ao criar um obj específico
    uma pessoa por exemplo, nao precisa criar primeiro o conceito daquilo. o obj 
    já é a propria instância dele

        uma função start() para emular o conceito do constructor. 
        o nome disso teórico é bootstrap, inicializador.
        uma alternativa caso precise executar algo antes de qualquer coisa.

        function createPerson(name, lastName, age){
            return {
                name, 
                lastName, 
                age,
                getFullName() {
                    return `${this.name} ${this.lastName}`;
                },
                start() {
                    console.log('deu start na pessoa');
                }
            }
        }

        let person1 = createPerson('Boniek', 'Lacerda', 90);
        console.log(person1.getFullName());
        
        let person2 = createPerson('Jorge', 'Yoshii', 80);
        let person3 = createPerson('Emilia', 'Yoshii', 70);
